## 一、整体架构：三条数据链路

```
                               ┌─────────────────────────┐
                               │   品项毛利率 =            │
                               │   (收入-耗材-手工费)/收入  │
                               └────────────┬────────────┘
                                            │
              ┌─────────────────────────────┼─────────────────────────────┐
              │                             │                             │
              ▼                             ▼                             ▼
    ┌─────────────────┐          ┌───────────────────┐        ┌───────────────────┐
    │ 链路1: 收入链路   │          │ 链路2: 耗材成本链路 │        │ 链路3: 手工费链路   │
    │                 │          │                   │        │                   │
    │ 核销记录主表     │          │ 耗材消耗明细表      │        │ 业绩流水+费用明细   │
    │ dm_opt_qy_user_ │          │ dwd_ord_order_qy_ │        │ ods_db_store_crm_ │
    │ execution_      │          │ execution_record_ │        │ per_income_detail │
    │ record_all_d    │          │ cost_all_d        │        │ + per_stats       │
    └────────┬────────┘          └─────────┬─────────┘        └─────────┬─────────┘
             │                             │                             │
             │  exe_income                 │  SUM(not_expand_            │  SUM(amount)
             │  exe_cnt                    │  exclude_tax_amount)        │  WHERE type=2
             │                             │  WHERE consumable_type      │
             │                             │  IN (1,3) AND status=1      │
             │                             │                             │
             └─────────────────────────────┼─────────────────────────────┘
                                           │
                                           ▼
                         ┌─────────────────────────────────────┐
                         │  dws_opt_qy_gross_margin_stats_all_d │
                         │  天 × 品项 × 门店 预聚合             │
                         │  gross_margin_amt = exe_income       │
                         │    - ware_cost - manual_fee          │
                         └─────────────────────────────────────┘
```

### 业务逻辑

```
每条记录的生成逻辑：

1. 从 dm_opt_qy_user_execution_record_all_d 取核销记录
   过滤：dp = T-1, is_valid = 1

2. 按 stat_date + product_id + tenant_id 聚合
   聚合前已关联：
   - 客户维度（customer_id → dim_user 取等级/渠道）
   - 门店维度（tenant_id → dim_tenant 取城市/大区）
   - 品项维度（product_id → dim_product 取赛道/标准名/营收分类）

3. 每条记录代表：某天 × 某品项 × 某门店的全部核销汇总
   不是单笔核销明细，是已聚合的统计行
```

### 使用时的关键注意事项

1. **聚合计算，不行级计算**：`SUM(gross_margin_amt) / SUM(NULLIF(exe_income,0))`，不能 `AVG(gross_margin_amt/exe_income)`
2. **dp 和 stat_date 的区别**：`dp` 是数仓分区日期（T+1），`stat_date` 是业务发生日期
3. **耗材只含高值**：与财务口径的差异根源（详见第五章）
4. **不含好物**：益生菌/卸妆膏等 22 个自营零售品核销营收=0，不在本表

#### consumable_type 枚举（毛利率的命门）

| 值 | 含义 | 毛利率是否计入 | 说明 |
|---|---|---|---|
| **1** | **设备耗材（高值）** | ✅ **计入** | 如热玛吉探头、超声刀刀头 |
| 2 | 低值耗材 | ❌ 不计入 | 如棉签、纱布、酒精 |
| **3** | **设备（高值）** | ✅ **计入** | 高值设备类耗材 |
| 4 | 设备 | ❌ 不计入 | 普通设备 |
| 5 | 普通商品 | ❌ 不计入 | 零售类商品 |
| 6 | 药品 | ❌ 不计入 | 如麻药 |
| 7 | 商品 | ❌ 不计入 | 通用商品类 |

> **这就是数仓与财务毛利率差异的根源**：
> - 数仓口径：只算 consumable_type IN (1,3)，高值耗材 = 收入中真正"烧掉"的核心成本
> - 财务口径：算全部 consumable_type 1-7，含低值耗材/药品/商品 = 完整成本
> - 差异：通常 1-4 个百分点

### 让利三公式的数据来源

```
① 价格让利率 = MAX(券前GMV - exe_amount - cash_back, 0) / (base_price × exe_cnt)
   ├── 券前GMV：base_price × exe_cnt（标价×服务点数）
   ├── exe_amount：实际成交金额
   └── cash_back：返现金额

② 用户运营额度率 = (split_new_coupon_deposit + split_coupon_deposit) / exe_income
   ├── split_new_coupon_deposit：新客券分摊额度
   ├── split_coupon_deposit：老客券分摊额度
   └── exe_income：核销收入

③ 门店改价额度率 = split_store_discount / exe_income
   ├── split_store_discount：门店改价分摊额度
   └── exe_income：核销收入
```

## 六、数据流全景图

```
                        用户到院核销
                             │
                             ▼
              ┌──────────────────────────────┐
              │ 院务系统记录：                  │
              │ · 做了什么品项                 │
              │ · 用了哪些耗材（品种/数量）      │
```

## 七、核心设计决策与已知问题

### 设计决策

| 决策 | 内容 | 影响 |
|---|---|---|
| 耗材口径 | 只含高值耗材 (consumable_type 1,3) | 与财务差 1-4pp |
| 成本价取值 | `not_expand_exclude_tax_amount`（不含税、不含销售公司分摊） | 直接=门店直采成本 |
| 聚合方式 | 必须 SUM 后相除，不能行级相除 | 1月15日修复 100%BUG |
| 手工费 | 只用 type=2（医护手工费），不拆分医生/护士 | 归因不够精细 |
| 好物排除 | 益生菌等 22 个零售品不进核销口径 | 医美 vs 零售口径隔离 |
