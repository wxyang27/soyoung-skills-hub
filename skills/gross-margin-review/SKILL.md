# 品项毛利率复盘分析

## 一、新人入门：什么是"品项毛利率"？

### 1.1 一句话定义

> **品项毛利率** = 核销收入扣掉直接成本后剩下的比例。
> 它回答一个问题：每收进来 100 块钱，去掉耗材和医生护士的操作费，还剩多少钱是毛利。

### 1.2 计算公式（必须记住）

```
品项毛利率 = SUM(毛利额) / SUM(NULLIF(核销收入, 0))

其中：
  毛利额 = 核销收入(exe_income) - 耗材成本(ware_cost) - 医护手工费(manual_fee)
```

**⚠️ 关键规则：必须用 SUM 聚合后再相除，不能逐行算完再取平均。**
这是 2026 年 1 月 15 日修复的一个 BUG——用行级计算会导致毛利率全部显示 100%。正确写法：

```sql
-- ✅ 正确
SELECT SUM(gross_margin_amt) / SUM(NULLIF(exe_income,0)) AS 毛利率
FROM soyoung_dw.dws_opt_qy_gross_margin_stats_all_d
WHERE dp = DATE_SUB(CURRENT_DATE(),1);

-- ❌ 错误（行级计算，毛利率全是 1）
SELECT gross_margin_amt / NULLIF(exe_income, 0) AS 毛利率
FROM ...
```

### 1.3 为什么它重要？

- **2026 年 2 月 7 日起纳入了公司 KPI 考核体系**——不是可选项，是必看指标
- 管理层每天看品项毛利日报（飞书机器人推送）
- 每月结账后财务对数，差异超过 1pp 就需要排查
- **目标值**：整体 48%，绿标品 ≥ 60-70%，常规品 ≥ 50%

## 二、毛利率由哪三部分构成？

每 1 元核销收入 = 耗材成本 + 医护手工费 + 毛利额。理解这三块是理解一切分析的基础。

### 2.1 核销收入（exe_income）—— "收了多少钱"

来源：用户到院做完治疗，核销单上的实际收入。

- 不是标价、不是支付价，是**核销当日的实际成交价**
- 可能被券抵扣、门店改价、历史订单价差等各种因素压低
- 包含 0 元单（赠品/体验单），现阶段 0 元单占比 18-24%

### 2.2 耗材成本（ware_cost）—— "用了多少耗材"

来源：每次核销，执行单上记录了用了哪些耗材、各多少。耗材消耗数据由吉客云/世纪通/ERP 出库记录同步。

**当前口径（2026.01.09 确认）：只算高值耗材，consumable_type IN (1,3)**

| consumable_type | 含义 | 是否计入毛利率 |
|---|---|---|
| 1 | 设备耗材（高值，如热玛吉探头、超声刀刀头）| ✅ 计入 |
| 2 | 低值耗材（棉签、纱布、酒精等）| ❌ 不计入 |
| 3 | 设备（高值）| ✅ 计入 |
| 4-7 | 设备/普通商品/药品/商品 | ❌ 不计入 |

**这就是数仓毛利率通常比财务口径高 1-4pp 的原因**——财务算全部耗材，数仓只算高值耗材。

### 2.3 医护手工费（manual_fee）—— "医生护士提成多少"

来源：CRM 系统记录的手工费/提成，按执行单 ID 关联。type=2 表示医护手工费。

- 不能拆分到医生 vs 护士（表结构限制，3 月 10 日确认）
- 规则相对固化可控，但与财务仍有约 2% 差异（原因是财务每月 28 日关账，月底数据跨期）
- 手工费率大盘约 7-8%，绿标品约 10-13%（因为治疗操作多），大师团仅 0.7-1.5%

## 三、三类品项分类规则（核心口径）

### 3.1 为什么是三类？

在 2026 年初，品项分类从原来的"大单品/常规品/大师团"演变为现在的"大师团/绿标品/常规品"三分法。关键变化是**"绿标品"（green label）取代了"大单品"概念**，且有明确的 32-35 个标准品项清单。2 月 4 日"品项毛利率"名称正式确定。

### 3.2 互斥优先级（必须严格执行）

```
输入：一条核销记录的 standard_name + revenue_category
         │
         ▼
    ┌────────────────────┐
    │ ① 是不是大师团？    │
    │ item_name LIKE      │──Yes──→ 大师团（结束）
    │ '大师团-%' OR       │
    │ revenue_category =  │
    │ '大师团'            │

            │ No

            │ No
            ▼
        常规品（兜底）
```

## 四、让利与补贴体系

### 4.1 核心概念：让利 ≠ 补贴

"让利"是更宽的概念，指标价与实收之间的差距。"补贴"是让利中的一部分（主要指券/红包驱动的让利）。

### 4.2 让利全景结构（基于 2026.01-05 实际数据）

```
核销收入 5.01亿
总让利 1.31亿（26.2%）

让利来源拆解：
┌──────────────────────────────────────────────────────────┐
│ 线上价格型 (On)  — 8.58%                                 │
│   ├── 促销补贴      5.89%（目标 5.16%）  ← 5月飙到 11.4%   │
│   ├── 老带新        2.40%（目标 2.45%）  ← 基本达标        │
│   ├── 用户运营      0.28%（目标 2.50%）  ← 低于目标        │
│   └── 促核销        ≈0%（目标 2.00%）                     │
├──────────────────────────────────────────────────────────┤
│ 线下价格型 (Off) — 7.97%（目标 2.49%！超 3 倍）           │
│   └── 门店改价      7.97%                                 │
├──────────────────────────────────────────────────────────┤
│ 历史价差         — 7.90%（自然消退中：13.8%→4.5%）        │
├──────────────────────────────────────────────────────────┤
│ 成本型           — 1.77%                                  │
│   ├── 灌券赠送      1.02%（509万）                         │
│   └── 门店开单赠送  0.75%（378万）                         │
└──────────────────────────────────────────────────────────┘
总目标让利 14.60% → 实际 26.2%（超 11.6pp）
```

### 4.3 让利三率公式

```
① 价格让利率    = MAX(券前GMV - exe_amount - cash_back, 0) / (base_price × exe_cnt)
   含义：标价与实际成交价之间的折扣率。券前GMV = base_price × exe_cnt

② 用户运营额度率 = (split_new_coupon_deposit + split_coupon_deposit) / exe_income
   含义：券/红包让利占核销收入的比例。split_new_coupon_deposit 是新客券、split_coupon_deposit 是老客券

③ 门店改价额度率 = split_store_discount / exe_income
   含义：门店现场改价让利占收入的比例。split_store_discount 是门店折扣分摊额度
```

**三率来源表**：`dm_opt_qy_user_execution_record_all_d`（134 字段的核销宽表）

### 4.6 L0 等级灌券黑洞

L0（最低会员等级）用户是补贴效率最低的群体：

- **发券 832 万张** → 用券人率仅 19.4%
- **补贴率 1,884%**：让利 709 万 > 核销收入 37.7 万（用券时等级口径）
- **纯薅率 68.9%**：核销用户运营券的人中，近 7 成 0 付费
- **0 元赠品成本 222.7 万**：占 L0 核销收入的 9.7%（当前等级口径）
- 补贴率随等级单调下降：L0(1884%) → L2(48.8%) → L5(32.5%) → L8(32.1%)

> 结论：灌券策略应向 L5+ 高等级倾斜，限制 L0/L1 的券额上限。

## 五、六步复盘法

当你接到"帮我复盘一下 X 月品项毛利率"的需求时，按以下顺序操作：

### Step 1：看总量

**要回答的问题**：大盘毛利率是多少？离目标（48%）差多少？月度趋势是上升还是下降？

**数据来源**：`dws_opt_qy_gross_margin_stats_all_d`，按 stat_date 月度聚合

**参考值（实际数据）**：
```
25.07: 41.5% → 25.10: 51.7% → 26.01: 48.7% → 26.03: 54.1%(峰值) → 26.04: 51.4% → 26.05: 49.0%
```

### Step 2：拆收入侧结构

**要回答的问题**：三类品（大师团/绿标品/常规品）各自贡献了多少收入？各自的毛利率是多少？谁的占比在变化？

**关键判断标准**：
- 绿标品收入占比上升 + 毛利率上升 = 结构性优化 ✅
- 常规品占比 65% 但毛利率 42% = 结构拖累 ⚠️
- 某类品毛利率突然跳变（如绿标品 3 月 73.6%→4 月 68.3%）→ 需要 Step 5 下钻

**输出格式**：
```
| 三类品 | 收入(万) | 占比 | 毛利率 | 较上月变化 | 诊断 |
```

### Step 3：拆成本侧

**要回答的问题**：耗材率和手工费率分别是多少？哪类品在改善/恶化？

**关键判断标准**：
- 耗材率下降 + 毛利率上升 = 成本优化有效 ✅
- 耗材率上升但毛利率不变 → 可能是低耗占比上升
- 0 元单中出现高值耗材 → 直接拖累毛利 🔴

**实际案例**：4 月绿标品耗材率 20.6%（历史最低），成本侧不是 4 月毛利率回落的真因。

### Step 4：拆补贴侧

**要回答的问题**：让利总比例是多少？促销补贴、门店改价、历史价差各占多少？有没有失控项？

**关键判断标准**：
- 促销补贴月度环比涨幅 > 3pp → 需要立即审查促销政策 🔴
- 门店改价 > 5% → 超目标 2 倍 ⚠️
- 常规品用户运营率 > 15% → 破红线 🔴

**实际案例**：4 月毛利率回落的真因是"让利侧三件事同时发生"——① 常规品价格让利率从 0.5% 跳至 5.6%（×10）、② 绿标品新品低毛利上市（奇迹胶原 22.5%）、③ 门店改价虽在收敛但绿标品仍 16.6%。

### Step 5：下钻异常到单品/门店/等级

当异常被定位到某类品后，继续拆：

1. **单品维度**：是哪个具体品项在拖累/拉升？用 pp 贡献度排序
   - 例：漾活光彩针 -2.46pp（销量从 482 万塌方到 215 万），奇迹胶原新品上市 -0.54pp（毛利率仅 22.5%）
2. **门店维度**：是哪些门店异常？成都晶融汇 158%、成都大悦城 -2.62%
3. **会员等级维度**：L0 补贴率 1884%、L1 补贴率 117%

### Step 6：交叉判断，给出结论

三个维度交叉验证：
- 毛利率↓ + 耗材率↑ + 补贴率↑ → 补贴驱动了高耗品放量，需审视补贴品项结构
- 毛利率↑ + 绿标品占比↑ + 耗材率↓ → 结构性优化，方向正确 ✅
- 毛利率↓ + 耗材率稳定 + 补贴率↑ → 问题在定价/让利侧，不是成本侧

**每个维度的 "4 月踩刹车清单" 示例（8 个品项）**：

| 品名 | 策略组 | 4月收入(万) | 毛利率 | 价格让利率 | 问题类型 |
|---|---|---|---|---|---|

## 七、常见数据问题与排查

### 7.1 已解决的 BUG（了解即可，不会再出现）

| 问题 | 发现日期 | 根因 | 修复方式 |
|---|---|---|---|
| 毛利率=100% | 1月15日 | 行级计算 | 改为 SUM 聚合 |
| 大师团毛利率 69%→40% | 1月21日 | revenue_category 标签维护错误 | 禾叶统一维护基础品标签 |
| 撤销订单无成本 | 1月26日 | status=2 的记录未入底表 | 补充 status=2 |
| 门店入库成本不准 | 1月26日 | 原系统字段不准确 | 调整为数仓加工逻辑 |

### 7.2 持续存在的问题（需要警惕）

| 问题 | 严重度 | 表现 | 排查方法 |
|---|---|---|---|
| **数仓 vs 财务口径差异** | 🟡 中 | 数仓毛利率比财务高 1-4pp | 确认是否用了 consumable_type IN (1,3)；财务算全量 |
| **门店毛利率异常** | 🟡 中 | 晶融汇 158%、大悦城 -2.62% | 逐单查耗材成本，看是否有单笔成本异常大 |
| **手工费无法拆分医护** | 🟢 低 | manual_fee 是医生+护士合并 | 表结构限制，无法解决 |
| **划扣记录关键字段=0** | 🔴 高 | 6月2日发现，order_id/product_id=0 | **研发正在紧急修复** |
| **耗材编码/单位不统一** | 🟡 中 | 大包装拆小无法算成本 | 等用友ERP上线（预计6月30日） |

### 7.3 三大系统性断裂点（影响所有毛利率分析的底层问题）

1. **品项映射断裂**：品项服务与耗材没有唯一关联（无"标准 SOP"），同一个品项不同医生可能用不同耗材
2. **计量单位断裂**：采购大包装（5ml/瓶），实际用小包装（1ml/次），成本差数倍，系统不支持自动换算
3. **源头成本断裂**：手工录入成本大量错误——门店入库贴码和手动录入的数据不可信

> 这三个问题预计随用友 ERP 上线（6月30日）逐步解决。
